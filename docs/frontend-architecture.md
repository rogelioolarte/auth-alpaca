## 🏗️ Angular 21 Architecture

The frontend is a single-page application built on **Angular 21** using standalone components. It manages async states and data streams with RxJS observables.

### Key Directory Structure
* **`src/app/auth/`**: Core authentication service, HTTP interceptors, routing guards, and redirection handlers.
* **`src/app/api/`**: API wrapper services (e.g. `AuthService`) communicating with the Spring Boot backend.
* **`src/app/models/`**: TypeScript interfaces, constants, and type models.
* **`src/app/components/`**: Reusable user interface components (e.g. login form, dashboard, layout).

## 🔒 Authentication Flow & HTTP Interceptor

Authentication is handled transparently across all HTTP requests using an Angular functional interceptor (`authInterceptor`).

### Interceptor Logic (`auth-interceptor.ts`)
The interceptor intercepts outgoing HTTP client requests and applies the following rules:
1. **Pass-through Rules**: Requests targeting authentication endpoints (`/api/auth/login`, `/api/auth/exchange`) or external hosts (e.g. Google APIs) bypass token injection.
2. **Token Injection**: Attaches the current active access token as a bearer token in the `Authorization` header (or configured header key).
3. **Automatic Token Refresh (411/401 retry)**:
   - If an API call fails with status `401 Unauthorized`, the interceptor halts the pipeline.
   - It triggers a token rotation request (`auth.rotateTokens()`) using the refresh token.
   - Upon successful rotation, the new access token is cached, the failed request is cloned with the updated token, and the request is retried.
4. **Session Invalidation**: If the API responds with a `430` error (typically signifying rate limit exhaustion or absolute session expiration), the client immediately invokes `auth.logout()` to clear client state.

## 🛡️ Routing Guards

Route access is protected by functional router guards defined under `src/app/auth/`:

1. **`authGuard`**: Protects secure routes by checking if the user is authenticated. If the active token is expired or missing, it blocks navigation and redirects the user to the `/login` route using an Angular `UrlTree`.
2. **`adminGuard`**: Restricts administrative dashboards to users who possess the `ADMIN` role in their access token claims.
3. **`ownerGuard`**: Secures resources that are contextually owned by the authenticated user (e.g., matching target resource ID with user principal ID).
4. **`externalGuard`**: Redirects authenticated users away from public pages (like login/register) to the dashboard.

## 💾 Session State & Token Storage

The application stores session tokens on the client side using cookies.

### Storage Strategy
- **Cookies**: Tokens are written via `ngx-cookie-service` with attributes configured for security:
  - `path: '/'`
  - `secure: true` (only transmitted over HTTPS)
  - `sameSite: 'Lax'` (mitigates CSRF vulnerability)
- **Token Keys**:
  - `accessToken`: Stores the JWT access token.
  - `refreshToken`: Stores the rotation refresh token.
  - `clientId`: Stores a unique client identifier (UUID v7 generated by the client) to track distinct browser sessions.

### Token Lifecycle & Auto-Refresh
The `AuthenticationService` tracks token expiration using a reactive model:
- The access token is decoded using `jwt-decode` to read the `exp` claim.
- The service initiates an RxJS `timer` set to trigger `3000ms` (3 seconds) prior to token expiration (`REFRESH_THREEHOLD_MS`).
- When the timer fires, a background token rotation is automatically requested to fetch a fresh token pair.

## 🚀 Run & Deployment Guides (Bun 1.3.11)

The frontend project utilizes **Bun 1.3.11** as its package manager and runtime environment, providing extremely fast installation and build execution.

### Prerequisites
Make sure Bun 1.3.11 is installed on your local system.

### Development Server
To install dependencies and start the development server:
```bash
# Install package dependencies
bun install

# Spin up Angular dev server on port 4200
bun run start
```
The server will run on `http://localhost:4200`.

### Production Build
To compile the production assets:
```bash
bun run build
```
The output directory will be populated under `dist/auth-alpaca-ui/browser`.

### Local Production Server Preview
You can build and preview the application locally using a light server:
```bash
bun run start:dist
```

> [!NOTE]
> **No Unit Tests exist for the UI.**
> Because this frontend is designed as a direct integration example to demonstrate the Spring Boot OAuth2/JWT authentication flow, it does not include comprehensive frontend unit test suites. The `.spec.ts` files contain only boilerplate test skeletons.
