Privacy Policy

Auth Alpaca — Last updated: July 2026

Auth Alpaca is a demonstration and reference implementation project. It is not a commercial service, does not operate production infrastructure, and does not collect or process personal data for any business purpose.

1. What This Project Is

Auth Alpaca is an open-source example application that illustrates OAuth2 and JWT authentication patterns. Its sole purpose is educational — to provide a runnable reference for developers learning about secure authentication.

2. Data Collected

When you run this application locally or in your own deployment:

  • Credentials: OAuth2 tokens and session identifiers are stored in your own database and never transmitted to any third party other than the Identity Provider (e.g., Google) you explicitly configured.
  • No telemetry: The application does not include analytics, tracking, monitoring services, or any form of telemetry.
  • No cookies for tracking: HTTP-only cookies are used exclusively for authentication token storage — not for advertising, analytics, or cross-site tracking.

3. Third-Party Services

If you configure OAuth2 login with Google (or any other provider), authentication is handled by that provider's servers. Please refer to their privacy policy for how they handle your data. Auth Alpaca itself does not share data with any third party.

4. Data Storage and Retention

All data resides in the PostgreSQL database you deploy. Auth Alpaca has no cloud backend, no external database, and no centralized data store. You are the sole controller of any data you enter or generate while running the application.

5. Contact

This is a personal open-source project. If you have questions about this policy, please open an issue on the project repository.

This policy applies solely to the Auth Alpaca software. If you are viewing this page as part of Google's OAuth consent screen verification, note that this application is a demonstration project and does not process real user data in production.
